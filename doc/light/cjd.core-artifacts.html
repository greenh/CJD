cjd.core-artifacts

CJD data model for artifacts defined in clojure.core.

Defines a set of "artifacts" (in essence, data structures or object classes) that collectivly constitute a data model for what is, in effect, a very rough abstract syntax tree for a clojure file.

An "artifact" models a specific kind of Clojure entity, such as a function, macro, variable, protocol, record... Because the set of such entities is obviously extensible (starting with defartifact), it follows that the set of artifacts must be exensible as well.

Note that there are several additional extensos— ArtifactBase and HasCJDoc — that are incorporated by defartifact into every artifact. Similarly, the SubartifactBase extenso forms the basis for all subartifacts.

HasNamedSubartifactsdefprotocol in cjd.core-artifacts

Protocol implemented by artifacts that have subartifacts that engender names within their namespace.

For example, a protocol itself gives rise to a name within its namespace; but so do its methods. As a consequence a protocol artifact would implement HasNamedSubartifacts, and return a list of those names on request. A record implemeting the protocol, however, does not induce any new names, and therefore an artifact representign the record would not need to implement HasNamedSubartifacts.

Methods

(named‑subartifacts‑of [this])

Returns a list of named subartifacts.

Namespacedefartifact in cjd.core-artifacts

(Namespace [artifacts* uses])

Artifact representing a Clojure namespace.

In a canonical state of affairs, a Namespace object exists at the root of a rough abstract syntax tree of a Clojure source file. As such, most other artifacts (functions, defs, macros, etc., etc.) appear as children of the Namespace, and much of the CDJ processing is driven from here.

parse-functionaldefn in cjd.core-artifacts

(parse‑functional [form])

Parses a "functional" form, which includes artifacts like macros in addition to proper functions. Functional forms include at least one, and possibly multiple, sets of parameters.

parse-functional is called by the driver upon encountering a top-level functor that declares an artifact of appropriate type.

Arguments

form — The form to be parsed.

Returns

An artifact of the type defined by the driver-defined make-artifact function.
