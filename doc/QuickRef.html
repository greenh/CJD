CJD Quick Reference

Phrasing elements

Element

Description

@(i phrase)

Renders phrase in italics.

@(b phrase)

Renders phrase in bold.

@(c phrase)

Renders phrase as "code", e.g. in a monospaced font.

@(u phrase)

Underlines phrase.

@(del phrase)

Renders phrase as "deleted" text, generally shown as strikethrough text.

@(sc phrase)

Renders phrase in small capitals.

@(sup phrase)

Renders phrase as a superscript.

@(sub phrase)

Renders phrase as a subscript.

@(ns)

Inserts the formatted name of the current namespace.

@(name)

Inserts the formatted name of the artifact being commented.

@(arg arg-name)

Inserts the name of a specified argument, key, field, or option, formatted in a fashion appropriate to the name's context of definition. arg-name must be a name, and should be the name of an entity defined in the current scope.

Beware that arg and its synonyms are also used as the tags of flow elements. Thus, if arg-name is not a single form, CJD will interpreted the element as the corresponding flow element, and the resulting output will be… strange.

Alternate tags: field option opt key. In this context, arg and its alternate tags are all synonyms: any of them will work regardless of the kind of entity arg-name actually represents.

@(form form)

Inserts a formatted Clojure form.

@(fun param-vector)

Inserts a formatted form representing a function. param-vec is a vector containing binding forms, just as one would find in the declaration of a function. Thus, the binding forms in param-vec can be names of putative variables, vectors of binding forms, or sets of key/binding form pairs. CJD recognizes the :keys keyword  as well.

@(image image-url)

@(im image-url)

Inserts the image specified by image-url, which must be a string.

@(link linkable phrase?)

@(l linkable phrase?)

Inserts a link to the documentation of the entity specified by linkable, which must be a symbol resolvable in the naming context in which the comment occurs. If phrase is present, then it is used as the content displayed for the link.

CJD derives the URL associated with the link based on resolution of linkable. See below for details.

@(linki linkable)

@(il linkable)

Similar to @link, but attempts to shorten the linkable name to a form that would be allowed if linkable had been imported. Thus, @(linki java.util.ArrayList) will show only "ArrayList:" as the link text.

@(linkto url phrase?)

@(lt url phrase?)

Link to a the URL specified by url, which must be a string. If phrase is present, then it is used as the text displayed for the link.

@(target id)

Assigns id as the identity of the location within the documentation where the element occurs, allowing links to refer to that location.

For example, given an element @(target "xyz") at some point in the text, an element (in the same document) like @(linkto "#xyz") will produce a link that references that point in the text..

 

Flow elements

 

Element

Description

@(p phrase-or-flow*)

Paragraph. Content can be either phrasing content, or a sequence of flow elements.

@(ul list-item*)

Unordered (bulleted) list. list-item arguments should all be @li elements..

@(ol list-item*)

Ordered (numbered) list. Contents should be a list of @li elements.

@(li flow*)

List item. Only meaningful within a @ul or @ol element.

@(pre string)

Preformatted content. The contents of string (which should be a string) is inserted .

@(example string+)

Example display, used to illustrate REPL-based user interaction sequences. Arguments are a sequence of strings, each of which contains preformatted content.

CJD interprets the strings of the sequence as alternately representing user input, then REPL output, and adds formatting (including REPL prompts) to provide a simulation of what a user would expect to see during a REPL session.

@(returns flow+)

@(return flow+)

@(rv flow+)

Documentation for a return value.

 

@(arg name flow+)

Documentation for an entity named within the local context, such as an parameter, option, key, or field. 

 

@(field name flow+)

@(key name flow+)

@(opt name flow+)

@(option name
default‑value flow+)

Essentially identical to @opt, but adds a parameter to specify a default value.

@(popt name param

   flow+)

Documentation for a "parameterized option". This is mostly intended for documenting command-line style arguments that have parameters.

@(see flow+)

Supplies "see-also" information, e.g. references to other documentation, that relate to the artifact.

@(author flow+)

Adds the name(s) of the author(s) of the artifact.

@(since flow+)

Specifies information about the date or release of inception for the documented artifact.

@(deprecated flow+)

Indicates that the artifact is deprecated.

 

In-text keywords interpreted by CJD

 

Keyword      

Action

:nb

 

Causes CJD to suppress inserting space between two words, symbols, or elements that it would ordinarily separate.

:br

 

Causes CJD to insert space between two words, symbols, or elements in situations where it would normally run them together.
