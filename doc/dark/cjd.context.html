Defines a "context" object, a record containing miscellaneous information about the state of affairs currently in effect during CJD documentation generation processing.

The object's contents are situationally updated as it propagates throughout the CDJ comment-handling process.

The context object.

A Context object serves to aggregate all of the contextual information in effect at any time within a running instance of CJD. The specific content varies wildly; some fields are fixed throughout an execution, while others are set based on whatever happens to be going on at a specific point in program execution.

By convention all fields are initialized to null values, then initialized by cjd-gen (or elsewhere) using the supplied methods, as needed.

As is typical, we prefer methods to ad-hoc use of keywords for field access, and hence the record structure. A few more bugs wrung out at compile time...
