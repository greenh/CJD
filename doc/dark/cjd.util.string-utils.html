re-split-atdefn in cjd.util.string-utils

(re‑split‑at [re string])

Splits a string into a sequence of sub-strings based matches reported by a regular expression.

The RE must have a particular form, viz. it must have a capture group that _ends_ at the point where the split will be made. The capture group can capture any subset of the string up to the final character, as the entire substring following the prior split point (or the beginning of the string) will be included in output substring.

Note that unlike re-seq, re-split-at does not discard any of the input string's characters.

Arguments

re — The regular expression to use for splitting.

string — The string to split.

Returns

A vector of split strings.
