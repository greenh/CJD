(copy‑resource [resource‑fn])

Adds a request to copy a resource to the output directory.

CJD evaluates the requests when it's had a chance to fully establish its Context, and can pass it to requesting functions. This allows the functions to select resources appropriately, based, for example, on the "theme" requested by the user.

Arguments

resource‑fn — A resource specification function. This is a function of the form (fn [context]), where

context — The Context object currently in use.

Returns — A tuple of the form [doc‑name resource‑name], where

doc‑name — A string containing the name of the document (relative to the output directory) to which the resource will be copied.

resource‑name — The name of the resource to copy. For Java, this is as defined by Class.getResource.

Or, if nothing needs to be copied, nil can be returned.
