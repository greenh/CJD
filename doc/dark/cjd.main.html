cjd.main

Main function for command-line operation.

To run CJD from the command line, use: 

java -cp <classpath> cjd.main [option...] dest-dir source...
where:

Arguments

dest‑dir — The pathname of the output directory. If it doesn't exist, it will be created.

source... — One or more file or directory pathnames containing Clojure files to be included in the CJD documentation run. If a pathname designates a directory, that directory will be searched for files with names ending in .clj.

option... — A list of options, which can be a combination of:

‑‑all — Selects all recognized artifacts for inclusion, regardless of whether or not they have documentation.

‑‑css <css-file>[;<css-file>...] — A semicolon-separated list of alternative CSS file names to be used.

‑‑docstrings — Uses docstrings as a documentation source in addition to CJD comments.

‑‑filter <filter> — Specifies the name of a function to select artifacts for inclusion in the output.

‑‑help — Prints this documentation and exits.

‑‑index — Specifies the name of an index document relative to the directory specified by dest-dir. Defaults to index.html.

‑‑nogen — Inhibits HTML generation

‑‑noindex — Inhibits index generation.

‑‑overview <overview> — The name of a .clj file, the first CJD comment of which will be used as the summary statement for the generated documentation in the index document.

‑‑requires <ns>[;<ns>...] — A semicolon-separated list of namespaces containing extensions to the base CJD functionality.

‑‑showopts — Debug tool that prints a list of the options as extracted from command-line arguments.

‑‑theme <theme> — Specifies <theme> as the styling theme for generated output.

Current standard themes include:

light — Light backround, black text (the default).

dark — Black background, white text

‑‑throw — Causes any warning to generate an exception

‑‑title <title> — Use <title> as the title for the documentation.

‑‑v <vopts> — Sets output verbosity. <vopts> is a string of single-letter output selectors.

‑‑version — Prints the version string for CJD and exits.

See also

All options have direct counterparts in the options map parameter of cjd-generator; see that function for additional details.
